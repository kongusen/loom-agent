## 文档原则

- 页面围绕当前公开对象和真实模块组织。
- 所有页面尽量区分“当前实现”和“未来方向”。
- 对开发者有意义的入口优先写在前面。
- 真实能力以 `loom/` 和 `tests/` 为准。
