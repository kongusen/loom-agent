# 应用场景

Loom 适合那些“不是一轮 prompt 就能结束”的任务。

## 优先场景

### 1. 代码开发与重构

- 需要长期上下文维持。
- 需要多工具联动。
- 需要把复杂问题拆成多个子任务。

对应代码基础：

- `loom/tools/`
- `loom/orchestration/`
- `loom/context/`

### 2. 研究与分析

- 需要迭代式探索。
- 需要保留证据和结论的关系。
- 需要把临时发现转化成可继续使用的工作界面。

对应代码基础：

- `loom/memory/`
- `loom/context/`
- `loom/tools/knowledge.py`

### 3. 运行时后台或 Agent 服务

- 需要多会话、多任务和生命周期管理。
- 需要稳定的接口和句柄。
- 需要权限、策略、产物和事件流。

对应代码基础：

- `loom/api/`
- `loom/api/handles.py`
- `loom/api/events.py`
- `loom/api/artifacts.py`

### 4. 生态扩展型产品

- 需要 Skill、Plugin 或 MCP 集成。
- 需要第三方能力插拔。
- 需要外部能力的可控接入。

对应代码基础：

- `loom/ecosystem/`
- `loom/capabilities/`

## 不适合的场景

- 只做一次简单问答。
- 不需要工具和上下文管理。
- 不打算维护长生命周期 Agent。

这类场景可以直接用更轻量的封装，不必引入 Loom 的完整复杂度。
