# 运维与排障

这一页面向“系统为什么没按预期工作”的问题。

## 先看哪里

| 问题类型 | 优先排查位置 |
|---|---|
| 会话、任务、运行状态异常 | `loom/api/` |
| 上下文失控、压缩异常 | `loom/context/` |
| Agent 提前终止或不终止 | `loom/execution/` |
| 工具不可用或行为异常 | `loom/tools/` |
| 子 Agent 递归或协作异常 | `loom/orchestration/`、`loom/cluster/` |
| 权限、审计、拦截问题 | `loom/safety/` |
| 模型接入异常 | `loom/providers/` |

## 常见判断方法

### 1. 先区分是“设计未实现”还是“实现有 bug”

Loom 当前处于持续收敛期，很多现象不能简单归结为 bug，也可能是：

- 文档写的是设计目标。
- 代码只实现了基础骨架。
- 某个高级能力还没完全接上。

### 2. 再区分是“边界问题”还是“逻辑问题”

- 边界问题：模块职责放错了。
- 逻辑问题：同一职责内部实现错误。

### 3. 最后回到能力矩阵确认预期

如果某项能力在 wiki 中被标注为 `部分实现` 或 `设计目标`，就不应按“稳定特性”的标准去要求它。
