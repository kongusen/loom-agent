# Loom v0.7.0 完整接入方案 - 执行总结

## 一句话总结

**将 loom v0.6.6 的"半接线架构"升级为 v0.7.0 的"四公理完全集成架构"，删除所有向后兼容代码。**

---

## 核心问题诊断

### 当前状态（v0.6.6）
- ✅ 四公理的核心组件已实现
- ❌ 但未完全集成到主执行路径
- ❌ 旧代码和新代码并存（半接线）

### 具体表现

**公理一（上下文工程）**
- 问题：PartitionManager 存在，但 `_build_messages()` 手工拼接，只用 3/5 分区
- 影响：working/skill 分区未生效，heartbeat 依赖的数据缺失

**公理二（场景约束）**
- 问题：ConstraintValidator 存在，但工具执行绕过验证
- 影响：场景约束形同虚设

**公理三（信息增益）**
- 问题：`publish_with_gain()` 存在，但主路径用 `emit()`
- 影响：冗余输出未过滤

**公理四（自我进化）**
- 问题：E1/E2 工具定义存在，但 execute 是 `_not_implemented`
- 影响：进化机制无法使用

---

## 解决方案架构

### 设计原则
1. **完全集成**：所有公理机制成为主路径，不是可选钩子
2. **删除旧代码**：不向后兼容，彻底清理
3. **最小实现**：只写必需代码，避免过度工程

### 五个阶段

```
Phase 1: 公理一 → 统一分区组装
Phase 2: 公理二 → 约束前置验证
Phase 3: 公理三 → 全链路门控
Phase 4: 公理四 → 进化闭环
Phase 5: 清理 → 删除旧代码
```

---

## 关键变更点

### 1. Agent._build_messages() 完全重写
```python
# 之前：手工拼接
messages = [SystemMessage(...)]
messages.extend(history)

# 之后：统一分区组装
await self._update_all_partitions()  # 更新 5 个分区
full_context = self.partition_mgr.get_context()  # 统一组装
messages = self._context_to_messages(full_context)
```

### 2. ToolRegistry.execute() 集成约束
```python
# 之前：直接执行
result = await tool.execute(params, ctx)

# 之后：前置验证
if not constraint_validator.validate(tool_call):
    return "[CONSTRAINT_VIOLATION]"
result = await tool.execute(params, ctx)
```

### 3. 所有事件使用门控
```python
# 之前：无条件发布
await self.event_bus.emit(event)

# 之后：门控发布
await self._emit_with_gain(event, payload)
```

### 4. E1/E2 工具完全可用
```python
# 之前：占位实现
async def _not_implemented(...): return "not wired"

# 之后：真实 handler
async def write_memory_handler(params, ctx, agent):
    await agent.memory.l2.store(entry)
    return "Memory stored"
```
